BlackBoxScanner Documentation

Welcome to the BlackBoxScanner documentation page! This library enables you to explore the interpretability of large language models by analyzing their inner workings, such as likelihoods, tokens, and other intriguing details.

Purpose

The library aims to provide an accessible platform for researchers, academics, and enthusiasts to:

  • Investigate the interpretability of large language models.
  • Expand understanding of model outputs and internal decision-making processes.
  • Foster collaborative improvements and contributions through pull requests.

Features

1. GenerativeModelOutputs

This module dives into your large language model (LLM) to extract and analyze its internal behaviors. Below are its key methods:

sentence_log_likelihoods

This method outputs a ScannedOutputs object containing tokens and their respective negative log-likelihoods (NLLs). The closer the NLL value is to zero, the more likely the token is to be generated by the LLM.

Example:

context = "The furry pet is a ..."
output = {"dog": -90.556, "cat": -91.663}

Here, "dog" is more likely to be outputted by the LLM than "cat" given the context.

view_top_k

This method provides a simple view of the next top-k probable tokens the LLM might output for a given context. The number of tokens (k) is customizable.

Note: Currently, the method uses a greedy approach to outline the top tokens. Future updates may introduce additional parameter-based strategies.
